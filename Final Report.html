ManiMAtCha

Mesh-Level Differentiable Chart-Based Rendering

Abstract

We augment the MAtCha Gaussian chart-based pipeline with a differentiable mesh loss inspired by NeuManifold, enabling end-to-end optimization of mesh geometry and resulting in sharper, watertight reconstructions from sparse views. Our approach injects a marching-cubes-based regularizer every few iterations, improving geometry without significantly increasing training time.

Technical Approach

Our project builds upon MAtCha’s chart-based differentiable surface reconstruction pipeline by incorporating surface refinement strategies inspired by NeuManifold. The core idea is to improve mesh quality and geometry fidelity while reducing the number of required input images. We aim to bridge the gap between chart prediction and high-quality mesh extraction by adding a differentiable mesh loss to the pipeline, refining the surface representation without re-optimizing the chart parameterization.

Baseline: MAtCha Pipeline

MAtCha (Monocular Atlas Charting) generates multi-chart surface parameterizations from monocular RGB images. The method takes a sparse set of views, predicts depth maps via monocular depth estimation (DepthAnything), and generates RGB-depth tracks, which are backprojected into 3D space. The pipeline uses structure-from-motion (SfM) to align and place them into a common coordinate frame. The aligned depth maps allow the model to create local charts that approximate the surface manifold. Then it initializes Gaussian surface elements across the manifold and optimizes to refine both the geometry and color fields across the charts.

Our Extension: Differentiable Mesh Loss with Frozen Charts

While the original MAtCha paper focuses extensively on jointly refining the chart parameterization and surface Gaussians, we decouple these two processes. Specifically, we freeze the chart layout after the initial alignment and focus entirely on refining the Gaussian surface elements using mesh-level supervision. This idea draws inspiration from NeuManifold, which introduces mesh optimization within implicit surface representations. However, instead of working with neural fields, we directly optimize Gaussian primitives anchored on the precomputed charts.

This divergence from MAtCha allows us to introduce a differentiable Marching Cubes module, which converts the Gaussian field into a high-resolution mesh. We compute rendering losses using the differentiable mesh output, enabling gradients to backpropagate into the Gaussian parameters (position, scale, rotation). Our losses primarily include a combination of:

  • Photometric consistency loss across views
  • Silhouette loss using masks from input images
  • Chamfer distance between predicted mesh and ground-truth samples (when available)
  • Mesh regularization losses such as smoothness or normal consistency

This pipeline enables us to render the extracted mesh, compare it to ground-truth or image-space references, and update the surface Gaussians while keeping the chart parameterization intact.

Implementation Challenges

  • Scalability: The differentiable Marching Cubes step is memory-intensive due to the voxel grid resolution and gradient computation. Initial mesh updates took ≈1–2 hours on our local machines; we mitigated this by reducing voxel resolution and filtering the number of Gaussians.
  • Hole Artifacts: Freezing charts led to geometric misalignments and holes in extracted meshes. We are exploring mild regularization to encourage better Gaussian coverage without re-optimizing charts.

Unique Contributions and Decisions

  • We treat the chart layout as a fixed surface prior, decoupling chart prediction from surface refinement.
  • We integrate NeuManifold-style differentiable mesh losses to improve surface accuracy.
  • We introduce modular optimization, separating chart initialization from Gaussian refinement.
  • We developed a prototype visualization interface to inspect mesh changes interactively as Gaussian parameters vary.

Lessons Learned

  • Decoupling chart refinement from surface optimization simplifies the pipeline but can introduce stability issues in geometry alignment.
  • Differentiable mesh loss is a powerful supervision signal but significantly increases computational cost, requiring careful batching and memory management.
  • Freezing chart geometry enhances modularity and interpretability but demands robust initialization to avoid compounding errors.
  • Our modified pipeline offers a flexible framework for experimenting with alternative loss functions or optimization strategies on mesh outputs without re-training chart prediction.
