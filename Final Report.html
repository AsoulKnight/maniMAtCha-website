Mesh-Level Differentiable Chart-Based Rendering

Abstract

We augment MAtCha’s Gaussian chart pipeline with a differentiable mesh loss inspired by NeuManifold, achieving sharper, watertight reconstructions from sparse views with minimal overhead.

Technical Approach

We freeze chart layouts post-SfM alignment and refine only Gaussian surface elements via a differentiable mesh-based loss, bridging chart prediction and high-quality mesh extraction.

Baseline: MAtCha Pipeline

MAtCha predicts per-view depth, aligns charts via SfM, initializes anisotropic Gaussians, and optimizes geometry and color jointly.

Extension: Differentiable Mesh Loss

Every 5 iterations (λmesh = 5), we extract a mesh, render it, and backpropagate an MSE loss between rendered and ground-truth images into Gaussian parameters.

# Mesh regularization snippet verts, faces, colors = extractor.extract_mesh_unbounded2(resolution=512) result = render_mesh_with_pytorch3d(mesh, cams, 0) mesh_loss = torch.mean((result['rgb'] - gt_image) ** 2) total_loss += total_regularization_loss + 5 * mesh_loss

Implementation Challenges

  • High memory & compute: mitigated via lower resolution and cache clearing.
  • Hole artifacts: exploring lightweight Gaussian coverage regularizers.

Contributions & Lessons

  • Decoupling charts and Gaussians simplifies design but can cause alignment instabilities.
  • Mesh loss boosts accuracy but adds overhead; effective batching and GPU memory management are key.
